# Challenge MottuFlow

Projeto desenvolvido como solução para a empresa **Mottu**, com foco na construção de uma aplicação para **gestão e organização de pátios de motos**, auxiliando no controle operacional, visualização e otimização do fluxo de veículos.

## Objetivo

O sistema tem como objetivo **facilitar o gerenciamento de motos em pátios logísticos**, permitindo registrar, visualizar e controlar o fluxo de entrada, saída e localização dos veículos dentro do pátio.

A proposta busca melhorar a **organização operacional**, reduzir erros no controle manual e fornecer uma **interface digital para acompanhamento das operações**.

## Funcionalidades Principais

- Cadastro de motos no sistema  
- Controle de entrada e saída de veículos  
- Visualização do estado atual do pátio  
- Organização das motos por posições ou setores  
- Processamento de imagens para identificação de motos no pátio  
- Estrutura preparada para expansão de funcionalidades

## Tecnologias Utilizadas

- **React Native**
- **TypeScript**
- **Java**
- **Spring Boot**
- **Banco de dados relacional**
- **Python**
- **Open CV**

## Estrutura do Projeto

O repositório contém a implementação da aplicação dividida em camadas principais:

- **Frontend** – Interface do usuário e interação com o sistema  
- **Backend** – Lógica de negócio e gerenciamento dos dados  
- **Banco de Dados** – Persistência das informações do sistema  
- **Visão Computacional** – Processamento de imagens utilizando Python e OpenCV para identificação de motos

Essa separação permite melhor manutenção, escalabilidade e organização do código.

## Contexto do Desafio

O projeto foi desenvolvido como parte de um **desafio acadêmico**, simulando um cenário real de logística para empresas que trabalham com grande volume de motos em pátios operacionais.

A solução proposta demonstra conceitos de:

- desenvolvimento de aplicações completas (full-stack)
- organização de dados logísticos
- criação de interfaces para operação em tempo real
